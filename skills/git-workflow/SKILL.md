## What I do
- Guide conventional commit message formatting
- Manage feature branches, rebasing, and merge strategies
- Create PRs with proper descriptions and review checklists
- Handle merge conflicts with minimal disruption
- Create releases, tags, and changelogs
- Manage git hooks and pre-commit checks

## Commit message format
```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

Types: feat, fix, refactor, test, docs, chore, perf, ci, style

## Branch workflow
1. Create feature branch from latest main/master
2. Make atomic commits with clear messages
3. Rebase on main before PR to keep history clean
4. Squash fixup commits before merge

## PR template
- What: Brief description of changes
- Why: Problem being solved or feature being added
- How: Key implementation decisions
- Testing: How changes were verified
- Screenshots: If UI changes
